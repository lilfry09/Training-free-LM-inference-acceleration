# 语言模型高效推理大作业执行计划

## 1. 目标拆分

本次大作业分为两条并行主线：

- 个人部分：复现一种无训练推理加速方法，在 `Pythia-70M` 上完成 `ppl` 和速度测试。
- 小组部分：集成多种推理加速方法，完成对比实验、分析讨论，并使用 `NeurIPS` 模板撰写英文论文。

当前建议优先实现两种方法：

- `GQA`：作为结构改造类方法，适合分析注意力头分组后对速度、显存和精度的影响。
- `FlashAttention`：作为算子级优化方法，适合做工程加速基线，也便于与普通 attention 直接对比。

## 2. 项目目录

建议统一使用以下目录：

- `models/`：下载的模型文件
- `data/`：下载的数据集缓存和导出样本
- `src/`：核心实现代码
- `scripts/`：下载、评测、测速脚本
- `outputs/`：实验结果、日志、图表
- `notes/`：阅读笔记、论文草稿、错误分析

## 3. 执行阶段

### 阶段 A：环境与基线

目标：

- 跑通原始 `Pythia-70M`
- 固定评测数据和测速方法
- 记录 baseline 指标

任务：

- 下载 `EleutherAI/pythia-70m`
- 下载 `wikitext` 和 `pg19`
- 编写统一评测脚本，输出：
  - `ppl`
  - `TTFT`
  - `TPOT`
  - `Throughput`
  - 可选的显存峰值

交付物：

- baseline 结果表
- 一份可以复现实验的运行命令

### 阶段 B：FlashAttention 复现

目标：

- 将 baseline attention 替换为 `FlashAttention`
- 测试长短上下文下的性能提升

任务：

- 检查当前 `torch`、CUDA、GPU 与 `flash-attn` 兼容性
- 安装并验证 `flash-attn`
- 在 `Pythia-70M` 上跑通前向推理
- 与 baseline 对比 `ppl` 与速度指标

重点分析：

- `FlashAttention` 是否在小模型上仍有明显收益
- 序列长度变化时收益是否更明显
- 精度是否保持一致或近似一致

### 阶段 C：GQA 实现

目标：

- 在不训练的前提下，为 `Pythia-70M` 构造一个可运行的 `GQA` 版本
- 明确这种近似改造带来的性能和精度变化

任务：

- 阅读 `Pythia-70M` 的 attention 结构
- 设计从多头 `K/V` 到分组共享 `K/V` 的转换方式
- 实现推理阶段 `GQA` attention
- 与 baseline、FlashAttention 版本做对比

风险提示：

- 严格来说，原始 `Pythia-70M` 并不是按 `GQA` 训练的，因此直接无训练改造可能带来较明显精度损失。
- 这不影响作业完成，反而可以作为分析亮点：为什么 `GQA` 对未训练适配模型的效果有限。

### 阶段 D：组合实验

目标：

- 测试 `GQA + FlashAttention` 是否可叠加
- 整理个人部分和小组部分可复用结果

任务：

- 组合运行 `baseline / flash / gqa / gqa+flash`
- 在短文本与长文本上分别测试
- 形成统一结果表和图

推荐实验矩阵：

- 数据集：`WikiText-2`、`PG-19` 单样本长文本
- 序列长度：`512`、`1024`、`2048`、`4096`
- 方法：`baseline`、`flashattention`、`gqa`、`gqa+flashattention`

### 阶段 E：论文与答辩材料

目标：

- 完成英文短论文
- 准备答辩展示材料

论文结构建议：

- `Abstract`：一句话说明问题、方法、主要发现
- `Introduction`：为什么 LLM 推理加速重要
- `Method`：baseline、FlashAttention、GQA、组合方案
- `Experiments`：设置、指标、结果、误差分析
- `Conclusion`：结论与局限性

答辩展示建议：

- 1 页任务定义
- 1 页方法概览
- 2 页实验结果
- 1 页失败案例或负结果分析
- 1 页结论与分工

## 4. 推荐推进顺序

建议按下面顺序推进，最稳：

1. 跑通 baseline
2. 先做 `FlashAttention`
3. 再做 `GQA`
4. 最后测试 `GQA + FlashAttention`
5. 补论文和答辩稿

原因：

- `FlashAttention` 更像工程优化，通常更容易先拿到稳定正结果。
- `GQA` 的无训练迁移风险更高，适合在 baseline 和 FlashAttention 稳定后再做。

## 5. 本周可执行清单

如果按最近一周推进，建议这样分配：

1. 第 1 天：下载模型和数据，跑通 baseline
2. 第 2 天：完成统一评测脚本
3. 第 3-4 天：完成 `FlashAttention` 实验
4. 第 5-6 天：完成 `GQA` 实现与实验
5. 第 7 天：整理结果表和论文初稿

## 6. 当前结论

你现在的最佳切入点是：

- 用 `Pythia-70M` 建立可靠 baseline
- 把 `FlashAttention` 作为第一个稳定方法
- 把 `GQA` 作为带分析价值的改造方法

这样既容易做出结果，也方便写出一篇有实验和讨论的课程论文。
